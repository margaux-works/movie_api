API Documentation

The myFlix API is a RESTful service that provides access to a rich database of movie information, including details on movies, directors, and genres. It also offers user management features, enabling developers to integrate user registration, profile updates, and favorite movie lists into their applications.

URL Endpoints

Request URL HTTP Method Query Parameters Request Body Response
Get a list of ALL movies /movies GET n/a None A JSON object holding data about all movies
Get data about a single movie by its title /movies/:title GET title None A JSON object holding data about a single movie, including a title, description, genre, director and an image URL.
Get data about a genre by name /movies/genre/:genreName GET genreName None A JSON object holding data about a genre, including genre name and description
Get data about a director by name /movies/directors/:directorName GET :name None A JSON object holding data about a director, including name, bio and birth year.
Register a new user /users POST n/a A JSON object holding data about the user who registered.
i.e { "name": "Leo", "email": "leodc@gmail.com", "favoriteMovies": [] } 		A JSON object holding data about the user who register, including: name, email, favourite movies amd id.
Update user's info (username) /users/:userId PUT userID None A JSON object the updated data about the user.
Add a movie to favourite list users/:userId/favorites/:movieTitle PUT user ID and movieTitle None A text message indicating the movie has been added to the favourite list.
`${movieTitle} has been added to user ${id}'s array`
Remove a movie from favourite list users/:userId/favorites/:movieTitle DELETE userID and movieTitle None A text message indicating the movie has been removed from the favourite list.
`${movieTitle} has been removed from user ${id}'s array`
Deregister existing user /users/:userId DELETE userID None A text message indicating the user has been deregistered.
`User ${id} has been deleted`
